# Categories

Rules for where content goes. Read this before filing anything.

## Routing Table

| Signal in content | Destination folder | Example |
|---|---|---|
| Code explanation, syntax, framework concept, language feature | `learning/` | "How React hooks work" |
| App idea, product concept, startup thought, feature idea | `ideas/` | "Uber for dog walkers" |
| Active project with deliverables, project context | `projects/` | "EnSocial animation revamp" |
| Markets, investing, financial models, valuation, accounting | `finance/` | "DCF for SaaS companies" |
| Tool, library, repo, service, resource, workflow tip | `tools/` | "ffmpeg cheat sheet" |
| Doesn't fit above OR ambiguous | `000-inbox/` | Anything unclear |

## Rules

1. When uncertain between two categories, pick the one where the user would look for it later.
2. If content spans two domains (e.g., a finance app idea), pick the PRIMARY intent. An app idea about finance goes in `ideas/`, not `finance/`.
3. Never create new top-level folders. If something doesn't fit, it goes in `000-inbox/`.
4. Read the content carefully before categorizing. A mention of "Python" doesn't automatically mean `learning/` — it could be a tool tip or a project note.
